Outliers¶

From the SPECIAL NOTES paragraph of the data set description (see link on the top), there exist some outliers we wish to identify. The description explicitly says "two of them are simply unusual sales ... I would recommend removing any houses with more than 4000 square feet from the data set (which eliminates these 5 unusual observations)". Thus, it makes sense for us to plot a graph of GrLivArea (which stands for "above ground living area square feet") against SalePrice.

Observation:¶

The target variable is right skewed. As linear models love normally distributed data, we want to transform this variable (using log transformation) and make it more normally distributed.

Remark. The log transformation can be used to make highly skewed distributions less skewed thus help make patterns in the data more interpretable and meet the assumptions of inferential statistics.

# Pool: NA means No Pool, thus we fill with None.
combine['PoolQC'].fillna('None', inplace=True)

combine["Alley"] = combine["Alley"].fillna("None")
combine["Fence"] = combine["Fence"].fillna("None")
combine["FireplaceQu"] = combine["FireplaceQu"].fillna("None")

# We process the rest according to the documentation
# Some of them are filled with None (like above)
# Some of the nulls means 0 (for example, GarageArea)
# Some of the nulls we want to fill with the mode or median
# Otherws might have special default values
# Read data description for the details

combine["LotFrontage"] = combine.groupby("Neighborhood")["LotFrontage"].transform(
    lambda x: x.fillna(x.median()))

for col in ('GarageType', 'GarageFinish', 'GarageQual', 'GarageCond'):
    combine[col] = combine[col].fillna('None')
    
for col in ('GarageYrBlt', 'GarageArea', 'GarageCars'):
    combine[col] = combine[col].fillna(0)
    
for col in ('BsmtFinSF1', 'BsmtFinSF2', 'BsmtUnfSF','TotalBsmtSF', 'BsmtFullBath', 'BsmtHalfBath'):
    combine[col] = combine[col].fillna(0)
    
for col in ('BsmtQual', 'BsmtCond', 'BsmtExposure', 'BsmtFinType1', 'BsmtFinType2'):
    combine[col] = combine[col].fillna('None')
    
    
combine["MasVnrType"] = combine["MasVnrType"].fillna("None")
combine["MasVnrArea"] = combine["MasVnrArea"].fillna(0)
    
combine['MSZoning'] = combine['MSZoning'].fillna(combine['MSZoning'].mode()[0])
    
combine = combine.drop(['Utilities'], axis=1)

combine["Functional"] = combine["Functional"].fillna("Typ")

Applying Box Cox Transformation of highly skewed features¶

The scipy function boxcox1p computes the Box-Cox transformation of 1+x. For further reading, check out this statistic webiste http://onlinestatbook.com/2/transformations/box-cox.html and also its documentation https://docs.scipy.org/doc/scipy-0.19.0/reference/generated/scipy.special.boxcox1p.html.

In [93]:
skewness = skewness[abs(skewness) > 0.75] # we want to transform the highly skewed features
print("There are {} skewed numerical features to Box Cox transform.".format(skewness.shape[0]))

from scipy.special import boxcox1p
skewed_features = skewness.index

l = 0.15
for feat in skewed_features:
    combine[feat] = boxcox1p(combine[feat], l)
